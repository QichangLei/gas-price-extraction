  Task 1: Gas Price Extraction Pipeline
                                                                                                                        
  Our team will build a complete Python pipeline that accepts gas station images or video clips as input and outputs
  structured fuel price records. The pipeline preprocesses each image — applying contrast normalization and
  region-of-interest cropping — before submitting batches of 15 images per API call to the Google Gemini multimodal
  model for OCR. The prompt enforces a strict output schema (fuel type, price, payment type, brand, confidence score)
  and explicitly handles the ⁹⁄₁₀ fraction notation standard in U.S. gas pricing (e.g., 2.99⁹ → $2.999). My individual
  contribution is the full implementation of this pipeline, including the preprocessor, Gemini API extractor, and the
  evaluation script that measures accuracy against the GPA4MME-v2 benchmark dataset. I will also fix the current
  evaluation naming-convention bug and rerun metrics. Success is measured by achieving an F1 score ≥ 0.85 and a Mean
  Absolute Error ≤ $0.01 on the GPA4MME-v2 test set of 20 labeled gas station scenes, across Regular, Mid-Grade,
  Premium, and Diesel grades.

  ---
  Task 2: Geo-Merging and Routing Integration

  Our team will bind each extracted price record to precise GPS coordinates and expose the combined dataset in a format
  that a routing algorithm can consume to make cost-efficient fuel stop decisions. The geo-merger joins OCR output with
  EXIF GPS metadata or a sidecar coordinate CSV, producing a timestamped record for every detection with fields: station
   location (lat/lon), fuel type, price, brand, payment type, and timestamp. This dataset is then used by a routing
  module that selects the cheapest available fuel stop within a configurable radius (default 10 km) along a given route,
   computing estimated fuel cost savings per trip segment using vehicle fuel consumption as an input parameter. My
  individual contribution is the geo-merger module (already implemented) and the routing query interface — a Python
  function that accepts a route as a sequence of waypoints and returns the optimal fuel stop with projected cost delta
  versus the most expensive nearby alternative. Success is measured by: (1) 100% of output records containing valid GPS
  coordinates when source imagery has location data, and (2) the routing module correctly identifying the cheapest
  station in a set of at least 10 simulated test scenarios with verified ground-truth pricing.

  Task 3: Intelligent Frame Selection for Dashcam Footage

  Our team will develop a frame selection strategy that filters dashcam video before OCR processing, reducing
  unnecessary API calls and storage overhead while preserving all frames that contain readable price information. The
  strategy operates in two stages: first, GPS proximity filtering discards frames captured more than 500 m from any
  known gas station in a reference database (sourced from OpenStreetMap); second, a lightweight visual classifier
  (MobileNetV3 or a Gemini-based scene description check) scores remaining frames for price-sign visibility, passing
  only frames that exceed a confidence threshold to the OCR pipeline. My individual contribution is the GPS proximity
  filter — a Python module that loads a station coordinate list, builds a spatial index (BallTree with haversine
  metric), and flags each video frame based on the truck's logged GPS track. Success is measured by two metrics: (1) a ≥
   60% reduction in frames forwarded to the OCR API compared to naive 1-fps sampling on a 10-minute test dashcam clip,
  and (2) a recall ≥ 0.90 on frames that contain a readable price sign as manually labeled in the same clip, ensuring
  high-value frames are not discarded.
